Rocket Launch

You need both operators to say launch before the launch actually happens

The wrong way

You may be tempted to solve this problem by first receiving one then the other.

The problem here is when Alice okay's the launch, but Bob hasn't yet. Alice should be able to change her mind, but she cannot. Imagine if she suddenly noticed a problem and wanted to abort the launch.

When using a double for, she can still change her mind because the double for won't consume her launch message until bob's message also appears and is ready to be consumed.

EXERCISE: Write the code that Alice would need to "par in" to retract her launch command.

Quiz

In for (x <- y; a <- b){ 0 }, which channel must be sent on first? y, b, doesn't matter, they must be sent simultaneously. In for (x <- y; a <- b){ 0 }, which message will be consumed first? y, b, doesn't matter, they will be consumed simultaneously.
